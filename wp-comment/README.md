# WP Comment Pins

A WordPress mini-plugin (part of SuiteWP) for a visual comment-pins system on
the front end — useful for editorial review and feedback. The front end is a
React app (built with `@wordpress/scripts`, reusing the React that WordPress
ships via `wp-element`).

## What makes it reliable

Pins are **anchored to DOM elements**, not to absolute page pixels. Each pin
stores a stable CSS selector for the element it was placed on plus an offset in
percent within that element's box. On load the pin is re-positioned from its
anchor, so it stays in place across screen sizes, scroll and content reflow.
(The previous version stored absolute coordinates, which drifted whenever the
layout changed — that was the root cause of pins "floating" off target.)

## Usage

Activate the **WP Comment Pins** module from the SuiteWP screen. A "Comment"
button appears in the admin bar for users with the required capability.

1. Click the "Comment" button to enter comment mode
2. Click anywhere on the page to drop a pin and write a comment
3. Click an existing pin to read it; the author (or a user with
   `edit_others_posts`) can delete it
4. Click the button again (or press Esc) to leave comment mode

## Security

- Gated by capability (`edit_posts` by default, filterable via
  `wp_comment_pins_capability`) at enqueue, admin bar and every AJAX endpoint
- Nonce-checked AJAX with consistent JSON errors
- Delete restricted to the pin's author or `edit_others_posts`
- Comment text is rendered through React (auto-escaped) — no `innerHTML`
- Pin URLs are normalised server-side under `home_url()`

## Database

Table `{prefix}comment_pins`:
`id, post_url, user_id, anchor_selector, offset_x, offset_y, comment_text, created_at`.

Created/upgraded through the SuiteWP lifecycle (`maybe_install_schema`,
version-gated via `wpcp_db_version`). Removed on uninstall only when SuiteWP's
"Delete all data" opt-in is enabled.

## Development

```
src/        # React source (JSX)
build/      # compiled bundle (shipped); generated by npm run build
package.json
```

- `npm install` — install build tooling (once)
- `npm run build` — compile `src/` → `build/`
- `npm start` — watch/rebuild during development

Only `build/` is loaded at runtime; `node_modules/` is dev-only and not needed
in production. The PHP enqueues `build/index.js` using `build/index.asset.php`
for its dependency list and cache-busting version.

> Requires WordPress 6.6+ (the bundle depends on the `react-jsx-runtime` script
> handle registered in 6.6).
